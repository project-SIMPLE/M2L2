# Configuring the Meta Quest 3 headsets

This guide provisions each of the six Meta Quest 3 headsets for the M2L2: initial setup, enabling developer mode, authorizing USB debugging from the Mac mini, and assigning each headset a fixed IP address that matches its sticker. Complete the [Mac mini](./macmini_configuration.md) and [router](./wifi_router_configuration.md) setup first. Repeat every step for all six headsets.

---

## 1. Initial setup

Run the standard Meta Quest 3 first-boot setup on each headset.

> During initial setup you can scan a QR code generated by the **Meta Horizon** app to speed things up. The code is not saved once you close the app, so take a screenshot of it beforehand and reuse the screenshot across headsets.

---

## 2. Enable developer mode

At the end of setup, enable developer mode for the headset from the **Meta Horizon** app (**Headset settings → Developer mode**). Follow Meta's official mobile device setup guide for the current procedure:

- <https://developers.meta.com/horizon/documentation/native/android/mobile-device-setup/>

---

## 3. Authorize USB debugging

1. Connect each headset to the Mac mini with a USB-C cable.
2. Accept the on-headset USB debugging prompt and choose to **always allow** debugging from this computer.
3. Launch an immersive application once to trigger and accept the data-collection prompt.

> ADB and the platform tools are installed on the Mac mini by the [install script](./installation_instructions.md).

---

## 4. Connect to WiFi and assign IP addresses

Connect each headset to the `M2L2_WIFI` network (see [router configuration](./wifi_router_configuration.md)), then give each headset a **static IP address** according to its sticker color.

The M2L2 is built around the TP-Link Deco's default subnet, `192.168.68.0/24`, so every device lives on `192.168.68.x`. Headsets use the `.101`–`.106` range, with the **last octet** mapping to the sticker color. If you change the router's subnet, adjust these addresses accordingly.

| IP address | Sticker color |
|---|---|
| `192.168.68.101` | Blue |
| `192.168.68.102` | Green |
| `192.168.68.103` | Orange |
| `192.168.68.104` | Purple |
| `192.168.68.105` | Yellow |
| `192.168.68.106` | Black |

The router's DHCP will **not** hand out these addresses on its own — each one must be set manually, using **one** of these methods:

- **From the headset (recommended)** — in the Quest WiFi settings, open the `M2L2_WIFI` network, switch the IP setting from DHCP to **Static**, and enter the address, gateway (`192.168.68.1`), and subnet mask (`255.255.255.0`).
- **From the TP-Link Deco app** — reserve the address against each headset's MAC (DHCP reservation / address binding) so the router always leases it the same IP.

> Use the headset method unless you have a reason not to: the IP travels with the headset and does not depend on the router remembering a reservation.

> You can enter text from the Mac mini instead of typing with the controllers using:
> ```bash
> adb shell input text "your text"
> ```
> This is much faster for entering WiFi passwords and account details on the headset.
